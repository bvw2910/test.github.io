fundamentos

< ! -- Introdução CSS -->

O CSS é a ferramenta que possibilita o ajuste estético da página através de comandos, interagindo com as marcações da arquitetura fornecida pela codificação em HTML. Os seletores utilizados pela linguagem são então combinados com as tags para a realização de funções específicas.



seleção por id

{

A função de seleção com o id é capaz de criar uma diferenciação na nomeação de tags do mesmo tipo, facilitando sua identificação e possibilitando formatações específicas para seu conteúdo. É uma função que exige exclusividade de nomenclatura para que não ocorra erros.Ex:
< h1 id="primeira-palavra >
** OBS: Não é possível nomear a id com espaços em branco, é costume utilizar o traço para isso **

}



seleção por classe

{

Através da seleção por classe é possível configurar formatações para um grupo de tags de mesma função, diminuindo o trabalho de repetir comandos iguais para estes.Ex:
< h1 class="title" >

}



< ! -- Mais Tags HTML -->

links html

{

HTML é uma linguagem de marcação criada para se comunicar com outras linguagens através do servidor, característica que o nomeia como Hiper Texto, pois se utiliza de hiper links para que essa comunicação ocorra.
A tag utilizada para direcionar ao link é a tag a de âncora, seguida por href="" de hiper referência, que contêm a URL absoluta (indicando o protocolo, domínio e local específico) ou URl relativa (indica o domínio e página específica, mas encontra-se no site em uso). Dentre essa tag e a de fechamento inclui- se o título do link. Outro recurso que pode ser utilizado é a ferramenta target="" para que o link abra em uma nova aba. Ex:

W3Schools | CSS Zen Garden

}



links internos html

{

Ir para o topo

< ! -- Propriedades de Texto CSS -->

font-family

{

Com o CSS a formatação das fontes tornam-se mais eficientes. É possível, por exemplo, amenizar as mudanças estéticas que ocorrem entre a máquina que desenvolve o site e a que vai acessá-lo, algo que ocorre devido a falta de alguns recursos em alguma das máquinas e gera diferença na aparência das fontes. Por conta disso, é um hábito entre aqueles que desenvolvem não somente especificarem a fonte que desejam usar como também a familia/grupo em que ela se enquadra (sans-serif, cursive, fantasy, monospace, etc).


  • Família Genérica - Um grupo de famílias fonte com uma estética similar (como Serif ou Monospace).
  • Família Fonte - Uma família tipográfica específica (como Arial ou Georgia).

Familia Genérica Familia Fonte Descrição
Serif Times New Roman/ Georgia Fontes serif possuem pequenas linhas no fim de alguns caracteres
Sans-serif Arial/ Verdana "Sans" significa "sem" - essas fontes não tem linhas no fim dos caracteres
Monospace Courier New/ Lucida Console Todos caracteres monospace possuem a mesma largura


}



font-size

{

O tamanho das fontes são medidos em pixels (px), no entanto, no caso dos títulos a medida é feita de modo relativo (em). Por exemplo, caso seja decidido que o tamanho da fonte usada nos parágrafos é de 11px e que a do título é de 2em, o navegador entenderá que o título deve ter 22px de tamanho.
Sites p/ Pesquisa: Google Fonts | Font Pair

}



herança

{

Herança diz respeito aos atributos que automaticamente são aplicados aos elementos que uma tag engloba. No entanto, caso haja necessidade de realizar alguma alteração em pontos específicos, há recursos que possiilitam a edição.

}

hospedagem site

{



Colocar um site "no ar" é o mesmo que, em outras palavras, hospedar uma página em um servidor. O servidor é um tipo de computador muito potente e conectado à internet, capaz de armazenar um número alto de páginas hospedadas e dispor ao usuário que o solicita através do acesso de uma url. Caso contrário, um código desenvolvido em um editor de textos só terá sua visualização possível no navegador da própria máquina ou com o envio de todos os arquivos. A esse tipo de acesso chamamos local de pois não pode ser compartilhado por um sevidor
Há muitas empresas hosts ou provedoras de hospedagem que fornecem esse serviço gratuitamente, como o Github e outras pagas, como a Amazon.

< ! -- Layout CSS -->

agrupamento de elementos

{



Para além da estilização, com o CSS é possível desenvolver o layout da página através do agrupamento dos elementos da mesma. Títulos, parágrafos, trechos de texto, imagens, podem ser agrupados e formatados conforme a necessidade. Isso é possível com o uso de duas tags: span e div.

< span >:

Agrupa alguns elementos de texto e dita sobre seu estilo, atua junto à uma classe em sua tag e de maneira inline [nesta página há diversos exemplos para ressaltar as tags e palavras-chave, basta simplesmente olhá-la como também inspecionar o código].

< div >:

Agrupa um conjunto de diversos elementos, utilizando um id para identificá-la e cria por fim, elementos em block [aqui no caso, por enquanto, há duas divs, uma para o título e outra para o conteúdo restante].
Com a div a página é dividida em diferentes blocos, que podem ser movidos e redimensionados.

}




largura, altura e overflow

{



< Largura > e < Altura >:

Após definir as divs é possível redimensioná-las na estilização do css atribuindo valores em pixels ou percentual conforme a necessidade do layout em criação.

< overflow >:

Essa propriedade surge quando há uma incompatibilidade entre as medidas atribuídas para altura e largura de uma div de texto, onde parte do conteúdo ultrapassa os limites estabelecidos, causando um desajuste visual. É um efeito que ocorre devido ao valor padrão do overflow ser visible, fazendo com que o texto apareça mesmo que as medidas da div sejam menores que o necessário.
O overflow, no entanto, pode assumir outros modos de atuação conforme o estabelecido na estilização do mesmo.

}




modelo de caixa

{



Cada elemento que compõe uma página web é lido pelo navegador como uma espécie de "caixa", na qual contêm valores da margem, borda, espaçamento e o próprio conteúdo que será exibido.



}




posição css

{



O navegador utiliza um posicionamento padrão chamado estático ou normal, ou seja, dispõe os elementos inline, como imagens, da esquerda para a direita e os em block, como cabeçalho e parágrafo, de cima para baixo. Para alterar esse padrão, utiliza-se a tag position.

< position >:

A tag position determina o tipo de posicionamento dos elementos na página. Existem três modos de operação da tag, que são: absolute, relative e fixed.

  • Absolute:

    • O modo absolute da a possibilidade de mover um elemento para qualquer lugar da tela. O valor absolute implica na sobreposição dos elementos a depender da ordem da codificação dos estilos. No entanto, há uma propriedade alternativa para lidar com casos do tipo, a z-index é capaz de ordenar a sobreposição dos elementos em questão.


  • Relative:

    • Ao usar o valor relative para a position, o navegador posicionará o elemento como normalmente faria, mas com alguma das quantidades alterada para deslocamento, podendo ser o top, right, bottom ou left.


  • Fixed:

    • O posicionamento fixed nos dá a impressão de que o objeto não se move, isso porque a opção o fixa em relação ao topo da janela da página.

}
